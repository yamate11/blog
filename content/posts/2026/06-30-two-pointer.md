### 概要

いくつか実装の選択肢があるので迷ってしまうが，次のように決めておくと良いかもしれない．

* ポインタは常に左から右に動かす．戻らない．
* 性質が成り立つ境界が2つあるわけだが，
  どちらの境界でも，静止状態では，ポインタは右側を指すようにする．
* ポインタが境界の左側を指していたときのなんらかの値が欲しいときには，
  それをどこかに記録しておく．

Rationale

* 戻すというのは常には可能ではないかもしれない．決して戻さないと決めておいた方が楽．
* ポインタを進めてみないと性質が成り立つかどうかわからないということがある．
* 左側の時の値が欲しい場合は，おそらく，少ない．

### 注意

つまり，「...がなりたつ最大のj」を求めようとしているときには，
「...が成り立たない最小のj」を求めることになるので，現在の j の値から 1 を引くことになる．

### 範囲右端に関する注意

「...が成り立つ最小のr」のタイプの場合，配列外参照などを避けるため，次のようなコードに
なることが多い:

```cpp
    while (r <= N and not cond(l, r)) r++;
```

この while ループを抜けたときは，r == N + 1 か cond(l, r) かであって，前者の場合には
cond (l, N) がなりたたなかったことを意味する．もっと大きな l についても同じことになるので
次のように書くことができる

```cpp
    for (ll l = 0; l < N; l++) {
        while (r <= N and not cond(l, r)) r++;
        if (r > N) break;
        do something with l and r
    }
```

「...が成り立つ最大のr」のタイプの場合，それは N だ，ということになる．not cond(l, r) で
抜けた場合も，r - 1 が「...が成り立つ最大の値」だし，r > N で抜けた場合も (r = N + 1 だから)
r - 1 が「...が成り立つ最大の値」になる．ということで，次のようなコードになる:

```cpp
    for (ll l = 0; l < N; l++) {
        while (r <= N and cond(l, r)) r++;
        // ここには break 文は無い
        do somethig with l and (r - 1)
    }
```

### 状況設定

$l \leq r$ なる整数 $l, r$ について定義されている述語 $P(l, r)$ で，次を満たすものを考える:

* $P(l, r)$ かつ $r \leq r'$ ならば $P(l, r')$
* $P(l, r)$ かつ $l' \leq l$ ならば $P(l', r)$

区間 $[l, r)$ に関する述語と見たとき，広いところの方が成り立ちやすい，ということ．
$\neg P$ を考えれば，狭いところの方が成り立ちやすい述語も扱える．
